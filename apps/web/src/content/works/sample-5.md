## 課題

学習塾向けの新規 SaaS をゼロから立ち上げる必要があり、要件は変動する一方で「同時接続 1 万人で破綻しない」という性能要件は固定。投資ラウンド前のため開発予算と期間が厳しく、外部パートナーへの委託は不可だった。

## 挑戦

DDD と Event Sourcing を組み合わせたアーキテクチャを採用し、学習履歴を不変のイベントストリームとして表現。スケジュールに縛られる中で「最初の 3 ヶ月は MVP」「次の 6 ヶ月で本番リリース」のフェーズ分けを徹底し、過剰投資を避けた。

## 解決

GCP Cloud Run + Cloud SQL + Pub/Sub のマネージドスタックで運用負荷を最小化。Astro + Tailwind の SSG で講師向けダッシュボードを高速配信。投資判断材料として Looker Studio で MAU / DAU / リテンションをリアルタイム可視化。

## 成果

| 指標                     | Before | After                                      |
| ------------------------ | ------ | ------------------------------------------ |
| MAU（リリース 6 ヶ月後） | 0      | 4.0 万                                     |
| 学習継続率（30 日）      | -      | 62%                                        |
| インフラ月額             | -      | $1,800（同時接続 1 万人想定の 1/4 で吸収） |
| 平均レスポンスタイム     | -      | 180ms                                      |

主要数値の要約：

- MAU: 0 → 4.0 万（リリース 6 ヶ月後）
- 学習継続率（30 日）: 62%
- インフラ月額: $1,800（想定の 1/4）
- 平均レスポンスタイム: 180ms
