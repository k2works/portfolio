# ADR-0003: MkDocs ドキュメントを「Tech Notes」として共存させ、初期は noindex で公開する

ポートフォリオ本体（Astro / `/`）と既存 MkDocs（`/docs/`）の関係を「採用ノイズにせず、技術的好奇心への補足リンクとして残す」位置づけで明文化する。

日付: 2026-04-30

## ステータス

2026-04-30 承認

## コンテキスト

- ヒアリング 3-B により、既存 MkDocs を残しつつ別途 Astro SSG をトップに配置する方針を採用済み（[ADR-0001](./0001-frontend-framework-astro.md)）。
- 設計レビューで以下の懸念が共通指摘された：
  - **PM**: 「ポートフォリオの中に学習ドキュメントが混じる」ことのメリットが説明されていない。戦略未決のまま技術判断（同居）が先行している。
  - **User Representative**: 「Docs」というナビ名から想像するのは「採用関連ドキュメント（職務経歴書 PDF 等）」であり、技術メモが出てきたら採用担当者が「迷い込んだ」感覚になる。
  - **Interaction Designer**: テーマ・レイアウト・ナビが切り替わり「別サイトに飛ばされた？」と混乱する可能性。
  - **Architect**: SEO 上、学習メモが採用検索の上位に出ると「ポートフォリオ」と誤認される。

検討した選択肢：

| 選択肢 | 概要 | 採用判断 |
|---|---|---|
| A. 完全削除 | MkDocs を Astro に統合、`/docs/` を廃止 | 既存資産（30 件のリファレンス・18 テンプレート）を放棄するコストが大きく不採用 |
| B. 学習継続アピールの主役級 | ナビから直接アクセス、検索インデックス対象 | 採用ノイズが大きく、訪問者の混乱を招く |
| C. **Tech Notes として補足リンクに留める** | ナビ名「Tech Notes」、フッター・Works 詳細から関連リンク、初期は `noindex` | **採用** |
| D. 完全に隠す | 直リンクのみ、ナビからも除外 | 学習継続性のアピールが完全に失われる |

## 決定

**選択肢 C を採用する。**

具体的な方針：

1. **ナビ名称**: ヘッダーナビでは「Docs」ではなく **「Tech Notes」** と表記する。アイコンに「↗」を付与し、訪問者に「別レイアウトのコンテンツへ」と予告する。
2. **役割分離の明文化**: `/docs/` トップに「これは個人の学習・設計メモです。職務経歴は Works を参照してください」というガイダンスバナーを置く。
3. **検索インデックス**: 初期は `<meta name="robots" content="noindex,nofollow">` を MkDocs ビルド時に挿入し、Google からの直接流入を抑制。サイト内回遊・直リンク経由のアクセスは妨げない。
4. **ブランド連続性**: ファビコン・サイト名・配色トークン（CSS カスタムプロパティ）を Astro と MkDocs で共有する。MkDocs 側のヘッダーに「← ポートフォリオに戻る」リンクを設置する。
5. **Works 詳細からの動線**: 各 Work の関連 ADR / 技術ノートに `/docs/...` への深いリンクを必要に応じて配置（同業エンジニア層への動線）。
6. **再評価**: 採用活動完了後または公開 1 年経過時点で、Google Search Console のデータを基に `noindex` 解除 / 完全公開 / 完全削除を判断する。

## 影響

### 良い影響

- 採用担当者が短時間で判断する際のノイズを抑え、Works への注意を集中できる。
- 同業エンジニア層には「興味があれば深掘りできる」豊富なリソースを提供できる。
- 既存 MkDocs 資産を捨てず、長期低頻度の更新で維持できる。
- ブランド連続性により「別サイトに飛ばされた」混乱を低減。

### 悪い影響・リスク

- `noindex` のため学習コンテンツが検索流入を生まない（採用には影響しないが、技術ブランディング機会を一部放棄）。
- MkDocs 側のヘッダー改修が必要（テーマカスタマイズ）。`mkdocs-material` の `extra_*` で対応可能だが工数が小さく発生。
- 「Tech Notes」と名付けても、訪問者の期待と中身（学習・設計メモ）にズレが残る可能性。

### 再評価のトリガー

- 採用活動完了後、または公開 1 年経過時点
- Google Search Console で「ポートフォリオ」と誤認するクエリ流入が観測された場合
- 学習コンテンツの SEO 機会損失が事業上の課題に発展した場合

## コンプライアンス

- MkDocs ビルド時に `robots` メタタグが `noindex,nofollow` を含むことを CI で確認（`grep` ベースのスクリプト or linkinator の追加検査）。
- ヘッダーナビの表記が「Tech Notes」かを E2E（E02）で検証する。
- `/docs/` のトップページにガイダンスバナーが存在することを E2E で検証する。
