# システムユースケース

## 概要

ポートフォリオサイトとアクターのインタラクションを「システムが提供する機能」の単位で記述する。各ユースケースは [ユーザーストーリー](./user_story.md) と E2E テスト ID（[テスト戦略](../design/test_strategy.md) E01〜E12）に紐づく。

## アクター

| アクター | 説明 |
|---|---|
| 訪問者 | 採用担当者 / 業務委託発注検討者 / 同業エンジニアなど、サイトを閲覧する人 |
| サイトオーナー | k2works 本人。コンテンツ追加・運用を担当 |
| クローラ | Googlebot 等の検索エンジン |

## 主要ユースケース詳細

### UC-01: プロフィールを閲覧する

| 項目 | 内容 |
|---|---|
| 主アクター | 訪問者（採用担当者、業務委託発注検討者、同業エンジニア） |
| 事前条件 | サイトに到達できる |
| 事後条件 | 訪問者が「誰がどんな専門領域か」を把握している |
| 主シナリオ | 1. 訪問者がトップページ `/` にアクセスする<br>2. システムは名前・役職タイトル・キャッチコピー・得意領域タグを表示する<br>3. システムは Featured Works 3 件のサマリを表示する<br>4. システムは Skills Highlights を表示する<br>5. システムはフッターに SNS リンクを表示する |
| 代替シナリオ | 2a. ダークモード設定がある場合: ダークテーマで表示する |
| 例外 | 1a. ネットワーク失敗: ブラウザ標準のエラー（404 / 500 はサーバ側で処理） |
| 受入条件 | E01: ホームに `<h1>` が存在し、Featured Works 3 件と Skills Highlights が表示される |

### UC-02: Works 一覧を閲覧する

| 項目 | 内容 |
|---|---|
| 主アクター | 訪問者 |
| 事前条件 | UC-01 完了、または検索からの直接流入 |
| 事後条件 | 訪問者が興味のある Work を選択している、または離脱している |
| 主シナリオ | 1. 訪問者が `/works/` にアクセスする<br>2. システムは Works カード一覧を表示する（カード: タイトル / 期間 / サマリ / 技術タグ / 「詳細」ボタン）<br>3. 訪問者は技術タグで絞り込む<br>4. システムは絞り込み結果を表示し、URL に `?tag=...` を付与する |
| 代替シナリオ | 3a. 「All」をクリック: 絞り込みを解除<br>4a. 結果 0 件: 「該当する Work がありません + フィルタを解除」を表示 |
| 受入条件 | E03 + 0/1/N 件の境界値テスト |

### UC-03: Works 詳細を閲覧する

| 項目 | 内容 |
|---|---|
| 主アクター | 訪問者 |
| 事前条件 | UC-02 のカード選択、または検索からの直接流入 |
| 事後条件 | 訪問者が「再現性のある成果か」を判断している |
| 主シナリオ | 1. 訪問者が `/works/{slug}/` にアクセスする<br>2. システムはパンくず（Home > Works > Work タイトル）を表示<br>3. システムは Work タイトル / 役職 / 期間 / カバー画像を表示<br>4. システムは「課題 → 挑戦 → 解決 → 成果」のストーリー構造で本文を表示<br>5. システムはチーム規模 / ポジション / 関与の深さ / 使用技術 / 外部リンク（Repo / Demo）を表示 |
| 例外 | 1a. 存在しない slug: UC-15（404）へ遷移 |
| 受入条件 | E04 + 外部リンクが `target="_blank" rel="noopener noreferrer"` を持つ |

### UC-05: 連絡先を確認する

| 項目 | 内容 |
|---|---|
| 主アクター | 訪問者 |
| 事前条件 | サイトのいずれかの画面に到達 |
| 事後条件 | 訪問者が連絡手段を選び、外部チャネルへ遷移している |
| 主シナリオ | 1. 訪問者がヘッダーの「Contact」または各画面の CTA をクリック<br>2. システムは連絡先案内（返信時間 + 稼働可否 availability）を表示<br>3. システムは連絡手段リンク（Email / GitHub / LinkedIn / X）を表示<br>4. 訪問者がリンクをクリック<br>5. システムは外部リンクを別タブで開く |
| 代替シナリオ | 2a. 稼働不可表示: 「現在新規案件は受け付けていません」と明示 |
| 受入条件 | E06 + availability の表示 |

### UC-08: コンテンツを追加・更新する（オーナー）

| 項目 | 内容 |
|---|---|
| 主アクター | サイトオーナー |
| 事前条件 | ローカル開発環境がセットアップ済み |
| 事後条件 | 新しい Work / Skill / Profile が production に反映されている |
| 主シナリオ | 1. オーナーが `apps/web/src/content/` 配下の Markdown を編集<br>2. ローカルで `npm run dev` で確認<br>3. PR を作成<br>4. CI（lint / test / build / E2E / Lighthouse）が成功<br>5. PR を main にマージ<br>6. GitHub Actions が staging に自動デプロイ<br>7. オーナーが staging で動作確認<br>8. 手動で production にプロモート |
| 代替シナリオ | 4a. CI 失敗: ローカルで再現・修正後再 push<br>8a. コンテンツのみの変更で Lighthouse 未達: ラベル `lighthouse-skip` で警告のみとする（[非機能要件参照](../design/non_functional.md)） |
| 受入条件 | Content Collections の Zod スキーマを通過、CI 全グリーン |

### UC-09: サイトの稼働を確認する（オーナー）

| 項目 | 内容 |
|---|---|
| 主アクター | サイトオーナー |
| 事前条件 | UptimeRobot が `/healthz` を 5 分間隔で監視 |
| 事後条件 | 稼働状況を把握、必要なら UC-10 を起動 |
| 主シナリオ | 1. UptimeRobot がメール / Pushover で通知<br>2. オーナーが Heroku ダッシュボードと Papertrail を確認<br>3. オーナーが障害種別を判定（SEV-1/2/3） |
| 例外 | 業務時間外は best effort |

### UC-10: 障害から復旧する（オーナー）

| 項目 | 内容 |
|---|---|
| 主アクター | サイトオーナー |
| 事前条件 | UC-09 で SEV-1/2 の障害を検知 |
| 事後条件 | サイトが復旧し、SEV-1/2 は Postmortem を起票 |
| 主シナリオ（Slug ロールバック） | 1. `heroku releases -a portfolio-prod` で過去リリースを確認<br>2. `heroku rollback v<N> -a portfolio-prod` で直前へ戻す<br>3. UptimeRobot で `/healthz` 復旧確認<br>4. Papertrail でエラーログ確認<br>5. Postmortem を起票 |
| 代替シナリオ | 直近デプロイが原因でない場合: `heroku ps:restart` を試す<br>Heroku Region 障害: Cloudflare DNS で GitHub Pages へ切替（[ADR-0004](../adr/0004-cloudflare-front-cdn.md)） |

## 横断的ユースケース（非機能）

### UC-11: 索引を生成する（クローラ）

| 項目 | 内容 |
|---|---|
| 主アクター | クローラ（Googlebot 等） |
| 主シナリオ | 1. クローラが `/sitemap.xml` を取得<br>2. クローラが各ページを取得<br>3. システムは `og:title` / `og:image` / `og:url` / `meta description` を返す<br>4. クローラが索引化 |
| 例外 | `/docs/*`: `<meta name="robots" content="noindex,nofollow">` のため索引化されない |
| 受入条件 | E10 + sitemap.xml の自動生成 |

### UC-12: ダークモードで閲覧する

| 項目 | 内容 |
|---|---|
| 主アクター | 訪問者 |
| 主シナリオ | 1. 初回訪問: システムは `prefers-color-scheme` を尊重<br>2. 訪問者がトグルクリック<br>3. システムは即時切替し `localStorage.theme` に保存<br>4. リロード後も同じテーマで表示 |
| 受入条件 | E08 |

### UC-15: 不明 URL からリカバリする

| 項目 | 内容 |
|---|---|
| 主アクター | 訪問者 |
| 主シナリオ | 1. 訪問者が存在しない URL にアクセス<br>2. システムは 404 ステータスで S90 を返す<br>3. システムは「ホームへ戻る」「Works を見る」の 2 動線を表示 |
| 受入条件 | E07 |
